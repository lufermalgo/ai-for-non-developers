# Contributing to AI for Non-Developers

Welcome! Whether you're a developer, a product visionary, or somewhere in between, we'd love your contribution.

## How to Contribute

### For Everyone: Ideas & Feedback

- **Found a problem?** [Open an issue](https://github.com/lufermalgo/ai-for-non-developers/issues/new?template=bug_report.md) with clear steps to reproduce.
- **Have an idea?** [Suggest a feature](https://github.com/lufermalgo/ai-for-non-developers/issues/new?template=feature_request.md) by describing what you want to achieve, not how to build it.
- **Documentation unclear?** Tell us where and how to improve it.

### For Developers: Code & Documentation

1. **Fork the repository** and create a feature branch: `git checkout -b feat/your-feature-name`
2. **Make changes** following the patterns in the codebase
3. **Test your changes** thoroughly
4. **Write clear commit messages**: `type(scope): what and why`
   - Examples: `docs(readme): clarify token efficiency`, `feat(cli): add validation`
5. **Open a Pull Request** with:
   - Clear description of what changed and why
   - How to test the changes
   - Link to any related issues
   - Evidence of testing (screenshots for UI, test results for code)

### For Product Owners: Shape the Vision

- **Validate specs**: We rely on your feedback to ensure the framework solves real problems
- **Test workflows**: Try the framework with your own projects and report what works (or doesn't)
- **Share your story**: Did this help you build something? We'd love to hear about it

## Code Standards

- Follow existing patterns in the codebase
- Keep code simple and readable
- No dead code or unnecessary complexity
- Prefer editing over rewriting
- Add comments only for the "why," not the "what"

## Review Process

1. Maintainers review your PR for alignment with project goals
2. We may ask for changes or clarifications
3. Once approved, your contribution is merged
4. You're celebrated! 🎉
